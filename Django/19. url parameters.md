## Project No: 0010
## Project Name: URLParamProject

**Objective**
This project focuses on understanding **Capturing Parameters from URL (Dynamic URLs)**.

---
# Step 1 — Create Project & App
```bash
django-admin startproject URLParamProject
cd URLParamProject
python manage.py startapp app1
```

---
# Step 2 — Register App
📄 URLParamProject/settings.py
```python
INSTALLED_APPS = [
    'django.contrib.admin',
    'django.contrib.staticfiles',
    'app1',
]
```

---
# Step 3 — Create View (App Level)
📄 app1/views.py
```python
from django.http import HttpResponse
def greet(request, name):
    return HttpResponse(f"Hello, {name}")
```

👉 Views should be created in **app level**, not project level (best practice)

---
# Step 4 — Create App URLs
📄 app1/urls.py  (create this file)
```python
from django.urls import path
from .views import greet

urlpatterns = [
    path('hello/<str:name>/', greet),
]
```

---
# Step 5 — Connect App URLs to Project
📄 URLParamProject/urls.py
```python
from django.contrib import admin
from django.urls import path, include

urlpatterns = [
    path('admin/', admin.site.urls),
    path('', include('app1.urls')),
]
```

---
# Step 6 — Run Server
```bash
python manage.py runserver
```

---
# Output
```
http://127.0.0.1:8000/hello/John/ → Hello, John
http://127.0.0.1:8000/hello/Ravi/ → Hello, Ravi
```

---
# URL Pattern Explained
```python
path('hello/<str:name>/', greet)
```

- `hello/` → static part
- `<str:name>` → dynamic parameter
- `name` → passed to view function

---
# Flow
```
URL → View → Parameter → Response
```
---
# Example
```
URL: /hello/John/
name = "John"
Output → Hello, John
```
